STEP 4: WHAT HAPPENS NEXT (Real-time flow)
---------------------------------------------

When you upload a video and a worker enters the zone:

1. Frame shows person approaching from top
2. Person crosses into zone (line 1)
3. ESP32 RFID is waiting
4. LED Status shows: 🟡 SCANNING RFID
5. You scan an RFID card
6. ESP32 receives UID → sends to Python
7. Python checks PPE (helmet + vest)
8. If SAFE (helmet YES + vest YES):
   ✓ LED Status shows: 🟢 SAFE
   ✓ Green LED lights up on ESP32
   ✓ Worker record saved to database
9. If UNSAFE (missing helmet or vest):
   ✓ LED Status shows: 🔴 UNSAFE
   ✓ Red LED + Buzzer on ESP32
   ✓ Warning record saved to database
10. Table updates automatically with new records
11. After 2 seconds, all LEDs turn off


REAL-TIME UPDATES

The frontend polls `/api/led-status` every 500ms to show:
  • Current status (idle, scanning, safe, unsafe)
  • Last scanned worker UID
  • Live LED emulation (Green/Red/Yellow indicators)
  • Status animations and glow effects


TABLE UPDATES

The "Worker Entry Logs" tab shows all scanned workers:
  • Worker ID (RFID UID)
  • Helmet status (Yes/No)
  • Vest status (Yes/No)
  • Timestamp

TROUBLESHOOTING

1. LED not showing in browser?
   → Check backend is running
   → Check browser console for errors
   → Rebuild frontend: npm run build

2. No worker records in table?
   → Check video uploaded successfully
   → Check person actually entered zone area
   → Check RFID card scanned (look for UID in ESP32 serial)

3. ESP32 not responding?
   → Check COM port in app.py (default: COM4)
   → Check serial connection at 115200 baud
   → Verify USB cable connected

4. Worker not detected?
   → Check YOLO model loaded (check console for "Loading model...")
   → Ensure video has clear person walking into zone
   → Check zone lines at 40% and 60% of video height


API ENDPOINTS

GET /api/led-status
  → Returns current LED status for real-time display

POST /api/process-video
  → Upload video for processing

GET /api/workers
  → Get all worker records

GET /api/status
  → Get summary (total, safe, unsafe counts)
