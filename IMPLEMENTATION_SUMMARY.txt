NEW FILES CREATED
════════════════════════════════════════════════════════════════════════════

1. python/app.py
   ─────────────────────────────────────────────────────────────────────
   Flask backend with integrated RFID + YOLOv8 system
   
   Features:
   • POST /api/process-video - Video upload & processing
   • GET /api/workers - Worker PPE records
   • GET /api/status - Status summary
   • YOLO frame-by-frame detection
   • Two-line zone crossing detection
   • RFID scanning with timeout
   • ESP32 signal transmission (G/R/O)
   • SQLite database integration
   • Gate assignment from filename
   
   Key Functions:
   • process_video_with_rfid() - Main processing pipeline
   • Zone locking logic - ONE person at a time
   • RFID wait-for-scan - 5-second timeout
   • PPE decision logic - helmet + vest → SAFE
   • Hardware signaling - Send to ESP32

2. python/detection.py
   ─────────────────────────────────────────────────────────────────────
   YOLO detection and two-line zone logic
   
   Classes:
   • PPEDetector
     - detect_frame() - Run YOLOv8 inference
     - is_contained() - Check if object within person bbox
     - get_center_y() - Get vertical center
     - crossing_zone() - Check zone entrance
   
   Functions:
   • draw_detection_results() - Annotate frame with results
   
   Output:
   • Bounding boxes (workers, helmets, vests)
   • Track IDs from ByteTrack
   • Zone lines visualization
   • Color-coded PPE status (GREEN/RED)

3. python/rfid.py
   ─────────────────────────────────────────────────────────────────────
   ESP32 serial communication
   
   Class: RFIDReader
   Methods:
   • __init__() - Connect to Serial port
   • connect() - Establish serial connection
   • read_uid() - Read RFID UID from ESP32
   • send_signal() - Send G/R/O commands
   • wait_for_rfid() - Wait with timeout
   • close() - Close serial connection
   
   Signals:
   • 'G' → Green LED (SAFE)
   • 'R' → Red LED + Buzzer (UNSAFE)
   • 'O' → Turn Off
   
   Features:
   • Automatic reconnection
   • Timeout handling
   • Error logging
   • Non-blocking reads

4. python/db.py
   ─────────────────────────────────────────────────────────────────────
   SQLite database operations
   
   Class: PPEDatabase
   Methods:
   • __init__() - Initialize DB connection
   • ensure_table() - Create worker_ppe table
   • insert_worker() - Save worker record
   • get_all_workers() - Retrieve all records
   • get_status_summary() - Get statistics
   • clear_records() - Reset before new video
   
   Schema:
   • worker_id (RFID UID)
   • helmet (YES/NO)
   • vest (YES/NO)
   • status (SAFE/UNSAFE)
   • gate (Gate1-5)
   • time (HH:MM:SS)
   • date (YYYY-MM-DD)

KEY FEATURES IMPLEMENTED
════════════════════════════════════════════════════════════════════════════

✅ VIDEO PROCESSING
   • Accept MP4, AVI, MOV, MKV, MPEG formats
   • Process frame-by-frame with YOLOv8
   • Detect: person, helmet, vest
   • Output: annotated MP4 with bounding boxes

✅ PPE DETECTION LOGIC
   • If helmet ∩ head AND vest ∩ body → SAFE (Green)
   • Else → UNSAFE (Red)
   • Show: ID, Helmet YES/NO, Vest YES/NO, Status

✅ TWO-LINE ZONE CONTROL
   • Draw horizontal lines at configurable heights
   • Detect when person enters zone (crosses line1)
   • Lock that person (ignore others)
   • Release lock after processing

✅ RFID INTEGRATION
   • Serial communication with ESP32 (115200 baud)
   • Read RFID UID when person locked
   • Timeout handling (5 seconds default)
   • Error handling for no scan

✅ DECISION & HARDWARE SIGNALING
   • Determine SAFE or UNSAFE
   • Send 'G' (green) if SAFE
   • Send 'R' (red + buzzer) if UNSAFE
   • Send 'O' (off) after 2 seconds

✅ DATABASE STORAGE
   • SQLite table: worker_ppe
   • Fields: worker_id, helmet, vest, status, gate, time, date
   • Insert records immediately after RFID
   • Gate assignment from filename (video1→Gate1, etc.)

✅ MULTI-GATE SUPPORT
   • Video1 → Gate1
   • Video2 → Gate2
   • Video3 → Gate3
   • Video4 → Gate4
   • Video5 → Gate5

✅ ERROR HANDLING
   • No RFID detected → Log and reset
   • Multiple people → Only process locked person
   • Serial disconnect → Graceful degradation
   • Missing model → Clear error message

✅ CODE STRUCTURE
   • Modular: app.py, detection.py, rfid.py, db.py
   • Clear separation of concerns
   • Comprehensive comments
   • Type hints for clarity
