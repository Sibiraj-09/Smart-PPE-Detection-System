🎯 SYSTEM FLOW (HOW IT WORKS):
────────────────────────────────────────────────────────────────────────────────

User uploads video
       ↓
YOLO detects people, helmets, vests
       ↓
Two-line zone detection locks one worker
       ↓
System waits for RFID card scan
       ↓

Records in database:
   • Worker ID (from RFID)
   • PPE status (helmet/vest)
   • Gate (from video name)
   • Time & Date (system timestamp)
       ↓
Dashboard shows results

🔧 HARDWARE CONFIGURATION PROVIDED:
────────────────────────────────────────────────────────────────────────────────

ESP32 Pin Connections:

RFID Module (MFRC522):
  SDA → D5      SCK → D18     MOSI → D23    
  MISO → D19    RST → D22     VCC → 3.3V    GND → GND

LEDs:
  Red LED long leg → D12 (+ 220Ω resistor to GND)
  Green LED long leg → D14 (+ 220Ω resistor to GND)

Buzzer:
  Positive (long pin) → D4    Negative → GND

Power:
  All VCC → 3.3V (breadboard red power line)
  All GND → GND (breadboard blue ground line)

📊 DATABASE SCHEMA:
────────────────────────────────────────────────────────────────────────────────

Table: worker_ppe

worker_id  ← RFID UID from scanner (primary field)
helmet     ← YES/NO (from YOLO detection)
vest       ← YES/NO (from YOLO detection)
status     ← SAFE/UNSAFE (determined by helmet AND vest)
gate       ← Gate1-5 (from video filename: video1=Gate1, etc.)
time       ← HH:MM:SS (captured at RFID scan)
date       ← YYYY-MM-DD (captured at RFID scan)

Example Record:
  worker_id: A1B2C3D4
  helmet: YES
  vest: YES
  status: SAFE
  gate: Gate1
  time: 14:35:22
  date: 2026-04-08

🔌 HARDWARE SIGNALS:
────────────────────────────────────────────────────────────────────────────────

Python sends 1-character commands to ESP32:

'G' (Green Command) - Send when SAFE status:
   → Green LED turns ON
   → Indicates worker has both helmet and vest

'R' (Red Command) - Send when UNSAFE status:
   → Red LED turns ON
   → Buzzer pulses: 0.5s ON, 0.2s OFF, 0.5s ON
   → Indicates worker missing PPE

'O' (Off Command) - Send after 2-3 seconds:
   → All LEDs turn OFF
   → Buzzer stops
   → System resets and waits for next worker
